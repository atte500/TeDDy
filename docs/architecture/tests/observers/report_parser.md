# Test Adapter: ReportParser (Inverse Adapter)
- **Status:** Planned

## 1. Purpose / Responsibility
Acting as the **Observer** in the Test Harness Triad, the `ReportParser` is a specialized test utility responsible for translating the Markdown Execution Report (generated by the system) back into structured data objects for easy assertion. This prevents tests from relying on fragile string matching and regex.

## 2. Ports
- **Primary Driving Adapter:** Used by the `CliTestAdapter` and test cases to verify outcomes.
- **Secondary Driven Port:** Uses the `IMarkdownPlanParser` infrastructure for consistent AST traversal.

## 3. Implementation Details / Logic
- **AST Mapping:** Maps Markdown sections (e.g., `## Action Log`) back to lists of `ActionData` and `ExecutionStatus`.
- **Typed DTOs:** Returns a `TestReport` object that mirrors the production `ExecutionReport` but is optimized for assertion ergonomics (e.g., `.action_was_successful(index=0)`).

## 4. Data Contracts / Methods
- `parse_from_stdout(stdout: str) -> TestReport`: Main entry point for CLI output.
- `extract_action_logs(stdout: str) -> list[dict]`: Helper to get the history of executed actions.
- `extract_resource_contents(stdout: str) -> dict[str, str]`: Helper to get contents of `READ` or `FAILED` actions.
